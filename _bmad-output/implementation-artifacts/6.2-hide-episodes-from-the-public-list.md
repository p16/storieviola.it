# Story 6.2: Hide episodes from the public list

## Status: done

## Story

As a **creator**,
I want **to mark an episode as `hidden: true` in its frontmatter**,
So that **I can keep an episode invisible on the site without deleting it**.

## Acceptance Criteria

**Given** an episode file with `hidden: true` in its frontmatter,
**When** the homepage renders,
**Then** that episode does not appear in the list, tag filter, or anywhere on the site.

**Given** an episode file with `hidden: false` or no `hidden` field,
**When** the homepage renders,
**Then** the episode appears normally — no change to existing behaviour.

**And** the `hidden` field is optional in the schema (boolean); omitting it is equivalent to `false`.
**And** the build fails if `hidden` is present but is not a boolean.

## Tasks

### Schema change — `src/content/schema.ts`

Add one field to `episodeEntrySchema`:

```ts
export const episodeEntrySchema = z.object({
  title: z.string(),
  description: z.string(),
  cover: z.string(),
  spotifyUrl: z.string(),
  publishDate: z.coerce.date(),
  tags: z.array(z.string()),
  hidden: z.boolean().optional(),   // ← add this
});
```

### index.astro change

Filter BEFORE collecting tags and sorting. Replace:

```ts
const allEpisodes = await getCollection('episodes');
const sortedEpisodes = sortEpisodesForPreview(allEpisodes);

const normalizedTags = allEpisodes
  .flatMap((ep) => ep.data.tags ?? [])
  ...
```

With:

```ts
const allEpisodes = await getCollection('episodes');
const visibleEpisodes = allEpisodes.filter((ep) => !ep.data.hidden);
const sortedEpisodes = sortEpisodesForPreview(visibleEpisodes);

const normalizedTags = visibleEpisodes
  .flatMap((ep) => ep.data.tags ?? [])
  ...
```

All downstream uses of `allEpisodes` in the frontmatter block must switch to `visibleEpisodes`. The template body below the frontmatter (`sortedEpisodes`) needs no change.

### No other files need changing

- `TagFilter.astro` — receives the already-filtered tag list; no change.
- `sortEpisodes.ts` — sort logic unchanged.

### Tests to add — `tests/unit/content-schema.test.ts`

Add inside the `episodeEntrySchema` describe block:

```ts
it('accepts hidden: true', () => {
  const parsed = episodeEntrySchema.parse({
    title: 'T', description: 'D', cover: '/x', spotifyUrl: 'https://open.spotify.com/episode/x',
    publishDate: new Date('2026-01-01'), tags: [], hidden: true,
  });
  expect(parsed.hidden).toBe(true);
});

it('accepts hidden: false', () => {
  const parsed = episodeEntrySchema.parse({
    title: 'T', description: 'D', cover: '/x', spotifyUrl: 'https://open.spotify.com/episode/x',
    publishDate: new Date('2026-01-01'), tags: [], hidden: false,
  });
  expect(parsed.hidden).toBe(false);
});

it('accepts omitted hidden (optional)', () => {
  const parsed = episodeEntrySchema.parse({
    title: 'T', description: 'D', cover: '/x', spotifyUrl: 'https://open.spotify.com/episode/x',
    publishDate: new Date('2026-01-01'), tags: [],
  });
  expect(parsed.hidden).toBeUndefined();
});

it('rejects hidden as a non-boolean', () => {
  expect(() =>
    episodeEntrySchema.parse({
      title: 'T', description: 'D', cover: '/x', spotifyUrl: 'https://open.spotify.com/episode/x',
      publishDate: new Date('2026-01-01'), tags: [], hidden: 'yes',
    })
  ).toThrow();
});
```
